# 驗車場智慧預約與流場管理系統

本專案目標是協助一般汽車檢驗場建立一套「線上預約、現場流場管理、車輛資料建檔、下次檢驗提醒」的整合系統。

## 專案定位

這不是單純的預約表單，而是驗車場每日營運用的中控工具：

- 車主可以查詢可預約時段、今日現場狀況與預估等待情形。
- 廠內人員可以管理預約車、臨時到場車、報到、等待、檢驗中、完成與覆驗狀態。
- 系統可以記錄車輛資料與下次檢驗日期，協助場內人員主動提醒車主。
- 後續可累積資料，用於尖峰分析、通知成效、回訪名單與營運決策。

## 目前成果

- `驗車場智慧預約與流場管理簡報.html`：可直接用瀏覽器開啟的網頁式簡報，方便與現場討論需求。
- `web/index.html`：第一版可操作的靜態 MVP 原型。
- `docs/需求規劃.md`：目前整理出的功能需求與模組規劃。
- `docs/現場訪談問題.md`：後續到現場訪談時可使用的問題清單。
- `docs/開發路線圖.md`：建議分階段導入範圍。
- `docs/開工紀錄.md`：記錄每次開工推進的範圍與決策。

## 安全提醒

後續若串接 LINE、簡訊、Email、資料庫或其他第三方 API，真實 API KEY、Token、密碼與私鑰不可寫進程式碼或提交到 Git。

- 本機請使用 `.env` 保存真實金鑰。
- 專案只提交 `.env.example` 作為欄位範本。
- 正式環境請使用部署平台的 Secret / Environment Variables。
- 詳細規範見 `SECURITY.md`。
