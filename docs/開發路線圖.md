# 開發路線圖

## 第一階段：MVP 基礎版

目標是先讓系統能被驗車場每天使用。

- 車主線上預約
- 時段容量設定
- 廠內預約管理
- 現場報到
- 車輛基本資料建檔
- 下次檢驗日期記錄
- 簡單提醒名單

## 第二階段：即時流場版

目標是把現場等待資訊同步給車主與廠內人員。

- 現場等待車數
- 檢驗流程狀態
- 臨時到場車管理
- 預估等待時間
- 現場大螢幕看板
- 車主端即時流量狀態

## 第三階段：通知與客戶經營版

目標是把車輛資料轉成可持續經營的提醒與回訪機制。

- 預約提醒
- 下次驗車到期提醒
- 逾期追蹤
- 批次通知
- LINE 或簡訊整合
- 未到率管理

## 第四階段：營運分析版

目標是讓驗車場可以用資料調整營運。

- 每日與每月驗車量
- 尖峰時段分析
- 平均等待時間
- 預約與臨時車比例
- 通知成效
- 回訪率
- 容量設定建議
