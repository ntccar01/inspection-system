# 開工紀錄

## 2026-06-23

本日正式進入開工狀態，先建立第一版可操作的靜態 MVP 原型。

## 本次開工範圍

- 建立 `web/index.html` 作為系統原型入口。
- 建立 `web/styles.css` 作為原型介面樣式。
- 建立 `web/app.js` 作為本機互動邏輯。
- 使用瀏覽器 `localStorage` 保存原型資料，不串接外部 API。
- 保持 API KEY 安全原則：本次沒有使用任何真實金鑰，後續串接第三方服務時需使用 `.env` 或部署平台 Secret。

## 第一版原型包含

- 今日看板
- 車主線上預約
- 時段容量顯示
- 廠內現場車流狀態更新
- 車輛資料與車齡判斷
- 下次檢驗提醒名單
- 時段容量設定

## 後續建議

1. 用此原型與現場討論流程是否符合日常作業。
2. 確認真正的時段容量、臨時車比例與狀態名稱。
3. 確認第一版要先做純內部系統，還是同步開放車主端頁面。
4. 決定後端、資料庫與通知服務導入時機。

## 2026-06-23 時段設定修正

- 將「時段容量設定」調整為「廠家時段設定」。
- 保留預設時段，但允許廠家自行新增、刪除與修改時段。
- 每個時段可設定開始時間、結束時間、線上預約上限與現場保留量。
- 修改時段時間後，既有預約會同步更新到新的時段名稱。
- 刪除已有預約紀錄的時段前，系統會先跳出確認提醒。

## 2026-06-23 車輛資料與工單分流

- 車輛資料表改顯示最近預約日、車齡、檢驗週期、下次建議日與通知同意。
- 新增「現場工單」分頁，集中顯示當日預約、報到、等待、檢驗中、完成與覆驗狀態。

## 2026-06-23 到期提醒清單修正

- 將「提醒名單」導覽改為「到期提醒」，頁面標題改為「驗車到期提醒名單」。
- 每筆提醒資料新增通知狀態，例如尚未通知、已通知、未接通、已回覆、不需通知。
- 每筆提醒資料新增後續處理，例如待處理、已預約、需再追蹤、暫緩、已完成。
- 每筆提醒資料新增追蹤備註，可記錄電話、LINE、簡訊或下次聯絡說明。
- 右側說明改為提醒處理方式，讓使用者能理解這頁是到期追蹤工作清單。
