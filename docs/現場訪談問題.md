# 現場訪談問題

這份問題清單用於與驗車場現場討論，確認第一版系統真正需要支援的流程。

## 營運與容量

- 每天平均驗車量約多少台？
- 最忙的月份、星期與時段是什麼？
- 每小時大約可消化幾台車？
- 是否需要保留名額給臨時到場車？
- 午休、設備保養、臨時停線時如何處理預約？

## 車主預約

- 車主目前主要透過電話、LINE、現場或其他方式預約？
- 預約時最少需要收集哪些資料？
- 是否允許車主自行取消或改期？
- 預約未到目前如何處理？
- 車主是否需要收到預約前一天提醒？

## 現場流程

- 車主到場後第一步是什麼？
- 報到由誰負責？
- 檢驗流程中有哪些重要狀態需要記錄？
- 檢驗完成後是否需要通知車主？
- 未通過與覆驗的處理流程是什麼？

## 車輛資料

- 目前是否有既有客戶或車輛資料？
- 車輛資料目前存在哪裡？
- 是否需要匯入舊資料？
- 下次驗車日期目前如何計算或記錄？
- 車主通知同意目前如何取得？

## 通知與回訪

- 想優先使用電話、簡訊還是 LINE 通知？
- 提醒車主的時間點希望提前多久？
- 是否需要批次產生提醒名單？
- 是否需要記錄通知是否成功或是否已回覆？

## 系統使用者

- 誰會使用後台？
- 櫃台、檢驗人員、管理者的權限是否不同？
- 是否需要每位員工自己的帳號？
- 是否需要操作紀錄？
